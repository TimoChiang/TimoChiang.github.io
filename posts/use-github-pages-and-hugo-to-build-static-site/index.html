使用GitHub Pages + Hugo搭建靜態網站

2020-03-29
GitHub
Comments

最近下定決心要開始寫技術文章，除了記錄下來自己的學習過程，也希望可以幫助到其他需要的人。

由於之前也有嘗試寫過但因為懶惰而中斷。期望這次可以好好持續下去！

正題開始

自己的需求取向如下：

  • 一個可以存放文章的地方（最好可以支援markdown）
  • 無廣告，而自己也沒有要從中獲取利潤（純粹紀錄＋分享）

原本打算在medium寫文章，但研究了一下，medium覺得不太符合自己的需求取向。

（medium比較適合從一開始就打算以類似出版的方式經營並從中獲取利潤的模式）

最後決定先嘗試用GitHub Pages + Hugo來紀錄自己的文章。

準備好Hugo

為何選擇Hugo?

Hugo是以Golang來快速生成靜態網站，而剛好最近也要學習Golang，於是沒多想就選擇他啦！

安裝Hugo環境

註：以下皆在macOS環境下執行。使用的Hugo Theme為hugo-theme-minos

首先依照官方網站的步驟安裝Hugo。（由於官方教學很清楚且步驟不多這裡就不贅述）

照著步驟安裝完後，應該已經在/content/posts/底下生成一個my-first-post.md文件。

並且已經成功啟動了Server，在http://localhost:1313看到已經搭建好的的網站。

編輯文章

除了建立一篇新文章之外，如何編輯文章也很重要。

這邊簡單說明一下如何修改標題與編輯內文。

在生成的檔案my-first-post.md中，可以發現已經有以下內容：

---
title: "My First Post"
date: 2020-03-29T16:51:38+09:00
draft: true
---
  • title: 文章標題
  • date: 文章的作成時間
  • draft: 是否為草稿，如果沒有加上-D 參數，草稿文章便不會顯示

這些是該文章的基本設定，而內文要在這個區塊底下開始。以下簡單用markdown來當範例：

---
title: "我的第一篇文章！"
date: 2020-03-29T16:51:38+09:00
draft: true
---
## 正文副標題

我是一段文字

- 我是清單一
- 我是清單二
- 我是清單三

部署至GitHub Pages

建立Github的repository

首先在GitHub上新增一個Github的repository，用來存放在Hugo生成靜態檔案。

官方教學

在建立之前有幾點要注意：

  1. 如果Github是使用免費方案，repository必須設為公開（如果希望是非公開的話，bitbucket有提供幾乎一樣功能且可以存放在非公開的repository）
  2. repository的名字必須是username.github.io，username需要跟GitHub的username一致，否則會無法成功部署。而這會是等等公開網站的預設domain。（有買網域可以自行更換）
  3. 如果GitHub的username有大寫，則會自動轉址到全部小寫的domain。例：username: AbcDefg -> domain: abcdefg.github.io

Hugo內設定git

接著回到Hugo資料夾，將本地的檔案連結至repository

在生成靜態檔案前，需要先在config.toml中設定好baseURL=username.github.io，否則會無法正常讀取靜態檔案。

生成靜態檔案，並設定git：

$ hugo #生成靜態檔案，預設是public資料夾
                   | EN
-------------------+-----
  Pages            |  6
  Paginator pages  |  0
  Non-page files   |  0
  Static files     | 26
  Processed images |  0
  Aliases          |  1
  Sitemaps         |  1
  Cleaned          |  0

$ cd /public
$ git init
$ git remote add origin git@github.com:username/username.github.io.git

接著把public底下的檔案push到username.github.io上，就完成部署啦！
